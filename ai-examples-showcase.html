学習システム

機械学習インタラクティブ教材
自動生成システム

「JSONファイルを書くだけで、Canvasアニメーション付きのインタラクティブHTML教材が完成する」仕組みを構築。 Kaggle・BERT・Python機械学習の3書籍12章以上を、Claude Codeとともに作り上げた。

何を解決したか

技術書の各章を教材化するとき、「内容を書く」より「ページを作る」作業のほうが圧倒的に時間がかかっていた。 コードハイライト・数式・インタラクティブデモを毎章手書きするのは現実的ではない。

そこで コンテンツ（JSON）とUI（Pythonテンプレ）を完全分離。 章の内容をJSONに書いたら、スクリプト1行でReact+KaTeX+Canvas付きのHTMLが秒単位で生成される。

  • React CDN + KaTeX + Prism.js + Canvas をすべて内包した単一HTML
  • スライダー・クイズ・グラフアニメーションをJSONから自動生成
  • テキストハイライト・付箋メモ・Gemini連携 (study-tools.js)
  • ダークモード / モバイル対応 / 進捗トラッカー搭載
  • gen_html.py 実行でトークン消費ゼロ、数秒で1章分を生成

開発フロー
1
PLAN
教材化する書籍・章を決める
「この技術書のどの章を教材化したいか」を決める。1章 = 1HTMLファイルなので、単元ごとに独立して作成できる。
2
EXTRACT
書籍PDFをGeminiに投げてJSONで網羅抽出
GeminiにPDFを渡して「この章の内容をJSONで出して」と頼む。 理論・数式・コード例・ポイント・注意点がまとめて出てくるので、それをそのままJSONファイルとして保存する。 PDFを丸ごと読んで構造化してくれるのがGeminiを使う理由。
3
GENERATE
gen_html.py で一発生成
python3 text_kaggle_win/gen_html.py text_kaggle_win/chapter-N.json
GeminiのJSONをそのまま渡すだけ。数秒・トークン消費ゼロ。React/KaTeX/Canvas入りの完全なHTMLが同ディレクトリに出力される。
4
PUBLISH
ダッシュボードにリンクを追加して完成
index.html のカードに章リンクを追加すれば即座に学習マップへ反映。内容を修正したいときはJSONを編集してgen_html.pyを再実行するだけ。
02
ナレッジ管理

Kindleハイライト →
「人生の示唆」7層分析

74冊・1,900件超のKindleハイライトをClaude APIで4段階に分析。 「読んで終わり」だった本棚を、自分の価値観や行動指針を映す鏡に変えるワークフロー。

何を解決したか

本を読むたびにハイライトが増えていくのに、半年後には何を読んだかも思い出せない。 74冊の学びがバラバラに存在していて、「自分が何を大事にしているか」が見えてこなかった。

Obsidianの自動同期でハイライトをMD化し、Pythonで束ねてからClaudeに渡す。 4段階の分析を経ると、読書の積み重ねが自分の価値観と行動指針になって出てくる。

① 手書きメモ全集
1,900件を本×タグ別に整理
② 価値観パターン分析
語彙のクセ・軸・葛藤
③ 横断テーマ集
著者を超えた共通洞察
④ 総合・人生の指針
行動指針10箇条 + 30日プラン

Team Insights

2つの事例から学べること

2つのプロジェクトを振り返ると、共通してやっていることがある。 Claude Codeに頼む前に、まず「仕組み」を作っているということ。

🔧
INSIGHT 01

「内容を書く」と「ページを作る」を
切り離す

ML教材では、chapter-N.json が内容、gen_html.py がHTMLの生成担当として役割が分かれている。 新しい章を追加するのは、JSONを書いてスクリプトを1行走らせるだけ。 この分離があったから、12章分の教材を無理なく作れた。

まず「型」を作る。内容はあとから流し込む。
📋
INSIGHT 02

CLAUDE.md に手順を書いておけば
毎回説明しなくていい

2つのプロジェクトとも、CLAUDE.md に「どう動くか」が書いてある。 Claude Codeはセッションのたびにこのファイルを読むので、「前回と同じように」が自動的に通じる。 カスタムスキル（kindle-summaryなど）は、その手順をコマンド1つで呼び出せるようにしたもの。

CLAUDE.md はチームへの引き継ぎ書でもある
📚
INSIGHT 03

AIが面白くなるのは
データが溜まってから

Kindleの分析が深いのは、74冊・1,900件という積み重ねがあるから。 AIは道具で、素材の質と量がそのまま出力に出る。 ML教材も章JSONが増えるほど学習マップとして使いやすくなる。 「何を貯めるか」を先に決めておくのが、どちらの事例にも共通している。
